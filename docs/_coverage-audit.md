## 3. Invariant

This audit is the drift-resistant guarantee that "the entire workflow is documented." To re-verify after adding/removing components:

```bash
npm run harvest          # regenerate component-inventory.json + gate-map
# then re-run the cross-reference: every inventory name must appear in
# docs/explanation/** OR be added to the catalog-only table above.
```

A new component that is neither named in an explanation doc nor listed catalog-only here is a coverage miss — document it (in the most-fitting explainer) or record it catalog-only with a reason.
