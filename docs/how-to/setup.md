# How-To: Install the Workflow on a New Machine

A task-oriented recipe for installing the full Claude workflow stack. For a guided first-run walkthrough, see [the getting-started tutorial](../tutorials/getting-started.md). For what each installed component does, see [the component inventory](../reference/component-inventory.md).

Agents, skills, and rules install at user level (`~/.claude/`) via symlinks, so they are available across all repos without per-project copies. Running `scripts/setup.sh` is the only manual install step.

---

## 1. Install prerequisites

| Tool | Install |
|------|---------|
| [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) | `npm install -g @anthropic-ai/claude-code` |
| [Node.js + npm](https://nodejs.org) | nodejs.org |
| [Python 3.11+](https://python.org) | Required by the `infra-init` skill's scripts (rejects < 3.11) |
| [uv](https://docs.astral.sh/uv/) | `pip install uv` |
| [pre-commit](https://pre-commit.com) | `pip install pre-commit` (per-repo pre-commit hooks) |
| Git | [git-scm.com](https://git-scm.com) (Git for Windows on Windows) |

**Windows note:** symlink creation requires Developer Mode (`Settings → System → For developers → Developer Mode`) or running the terminal as Administrator. `setup.sh` detects the platform and uses `ln -s` (Unix / git-bash) or `New-Item -ItemType SymbolicLink` (PowerShell); on failure it prints an actionable error.

---

## 2. Run the installer

```bash
git clone <repo-url>
cd claude-workflow-improvements
bash scripts/setup.sh
```

The script is idempotent (safe to re-run; existing symlinks are not replaced unless `--force` is passed). It:

1. Checks prerequisites (node, npm, python, pre-commit, claude CLI, git).
2. Creates the `~/.claude/` structure if missing.
3. Symlinks each agent file, skill directory, and rule file from the repo into `~/.claude/`.
4. Installs the global pre-commit hook (`hooks/pre-commit` → `~/.claude/hooks/pre-commit`, `chmod +x`).
5. Installs MCP npm packages globally and merges MCP entries into `~/.claude/settings.json` (additive — never overwrites existing entries).
6. Prints a summary of what was linked vs. already up to date.

The root `CLAUDE.md` is symlinked to `~/.claude/CLAUDE.md` (mandatory delegation + architect-review gates).

## 3. What gets installed

The authoritative, drift-checked roster of every skill, agent, rule, and hook is [docs/reference/component-inventory.md](../reference/component-inventory.md) (generated by `npm run harvest`). Do not hand-maintain a component list here — cite the inventory.

Component classes and their install targets:

| Class | Target | Reference |
|-------|--------|-----------|
| Agents | `~/.claude/agents/<name>.md` | [features/agents-and-skills.md](../explanation/features/agents-and-skills.md) |
| Skills | `~/.claude/skills/<name>/` | inventory |
| Rules | `~/.claude/rules/<name>.md` | [architecture.md](../explanation/architecture.md) § Instruction-loading layers |
| Hooks | `~/.claude/hooks/` (wired in `settings.json`) | [features/orchestration-gating.md](../explanation/features/orchestration-gating.md) |

**Install strategy: symlinks, not copies.** Any update committed to the repo is immediately reflected in `~/.claude/` — no re-run needed. To recreate all symlinks after adding new files: `bash scripts/setup.sh --force`.

**Plugins** (installed via `claude plugin install`) are tracked separately from the component inventory; see the README's Plugins section for the current set and `plugins/registry.md` for lifecycle state.

## 4. Configure MCP servers + credentials

`setup.sh` merges three MCP server entries into `~/.claude/settings.json` but does not handle secrets. After install:

1. Verify servers: `claude mcp list` — `bitbucket`, `atlassian`, and `git` should appear.
2. Authenticate Atlassian: `claude mcp auth atlassian`.
3. Add credentials per the [secrets-handling rule](../../rules/secrets-handling.md) — never paste a secret into the chat. Store via the OS credential manager, a permission-restricted `~/.netrc`, or shell-profile environment variables:

| Credential | Environment variable |
|------------|---------------------|
| Bitbucket username | `BITBUCKET_USERNAME` |
| Bitbucket API token | `BITBUCKET_APP_PASSWORD` |
| Atlassian API token | `ATLASSIAN_API_TOKEN` |
| Atlassian base URL | `ATLASSIAN_BASE_URL` |
| Atlassian email | `ATLASSIAN_EMAIL` |

> Bitbucket app passwords are being retired — use an API token paired with your Atlassian account email.

## 5. Verify the install

| Component | Verification |
|-----------|--------------|
| `git-manager` skill | `/git-manager commit files:[README.md] type:chore description:"test"` on a scratch repo — should stage + commit |
| `/infra-init` skill | run on this repo — should produce `.claude-init/CODEBASE.md` + `enrichments.json` |
| `/e2e-init` skill | run on a sample repo — should produce `.claude/testing-plan.md` |
| `architect` agent | invoke with any `plans/**/*-plan.md` path — should return BLOCKING / MINOR / APPROVED |
| MCP servers | `claude mcp list` — all three appear |

Once verified, start any session with `/using-superpowers` to initialize the orientation protocol.
