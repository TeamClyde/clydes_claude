# New Repo Setup

All agents and skills are global — they live in `~/.claude/` and are available in every repo without per-project copies. Install them once by running `scripts/setup.sh`.

---

## Agent/Skill Registry

### Agents — invoke via Agent tool (`subagent_type: <name>`)

| Agent | Role |
|-------|------|
| `architect` | Plan reviewer — returns `error` / `warning` findings + an `APPROVED` / `NEEDS REVISION` verdict before ExitPlanMode |
| `jira-workflow-manager` | All Jira operations: ticket creation, status transitions, updates |
| `researcher` | Codebase and infrastructure lookups — keeps search out of main context window |
| `test-strategy` | Derives per-plan validation criteria before implementation begins |
| `test-builder` | Writes test code from test strategy output, in parallel with implementation |
| `integration-engineer` | Maps cross-repo endpoints and contracts using local codebase MCPs |

**Spawned-only agents** (no persistent file — invoked at runtime by `infra-init` skill only):

| Agent | Role |
|-------|------|
| `infra-init-structure` | Phase 1: repo type detection and batch assignment |
| `infra-init-batch-indexer` | Phase 2: parallel source file extraction per batch |
| `infra-init-graph-builder` | Phase 3: queries codebase-memory-mcp and produces `CODEBASE.md` |

### Skills — invoke via Skill tool (`skill: <name>`)

| Skill | Role |
|-------|------|
| `git-manager` | All git operations: commits, branching, push, PR creation, worktree management |
| `infra-init` | Codebase graph generation — orchestrates the 3-phase infra-init agent workflow |
| `e2e-init` | Per-repo testing backbone setup — produces `testing-plan.md`, `e2e-plan.md`, and `run-tests.sh` |
| `plan-management` | TODO.md maintenance and plan doc status tracking |
| `creating-tools` | Orchestration entry point for all component creation — routes to correct skill by artifact type |
| `writing-agents` | TDD methodology for agent creation — baseline invocation required before writing system prompt |
| `writing-rules` | Guidance for creating rules files — rule vs skill decision, global vs path-scoped, observational testing |

## New Repo Checklist

Run these steps in order when onboarding any new repository.

1. **Run `scripts/setup.sh`** — symlinks global agents, skills, and rules into `~/.claude/`. Idempotent; safe to re-run.
2. **Run `/infra-init`** — required for repos with 200+ source files. Indexes repo via codebase-memory-mcp; generates `.claude-init/CODEBASE.md` and `.claude-init/enrichments.json`. Skip for small repos.
3. **Run `/e2e-init`** — generates `.claude/testing-plan.md`, `plans/e2e-plan.md`, and `scripts/run-tests.sh`. Run after `/infra-init` if both are needed.
4. **Create project `CLAUDE.md`** from the template below. Populate all placeholders before starting work.
5. **Declare stack hats (optional).** If the repo uses stacks with catalog entries in `~/.claude/stacks/` (e.g. `python`), add `"stacks": ["python", ...]` to `project.json`. The `sessionStart/stack-hat-directive.mjs` hook injects the matching `## Hat` guidance each session. No entry yet for a stack? The hook notes it harmlessly; author one from `stacks/_TEMPLATE.md`.
6. **Create initial `TODO.md`** — sections: `In Progress` / `Up Next` / `Backlog` / `History`.
7. **Configure pre-commit hook** — set `scripts/run-tests.sh` (generated by `/e2e-init`) as the pre-commit script. The global hook entry point is installed by `setup.sh`; per-repo configuration is handled by `/e2e-init`.
